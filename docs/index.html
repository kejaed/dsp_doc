Abstract

The DSP here is intentionally textbook — an LFM chirp, two-way spreading, matched filtering, range/Doppler, CA-CFAR. The point of this document is not the DSP. The point is that everything below is running in your browser: no server, no Jupyter kernel, no cloud function. The numerical code is real NumPy/SciPy executing inside Pyodide; the controls are wired through OJS reactivity; the figures redraw on each knob change in well under a second. The whole experience ships as a single self-contained report.html you can host on any static URL — or open from your local filesystem.

First load. Pyodide and its scientific stack are about 10 MB on first visit. Grab a coffee, then everything thereafter is local.

Setup

The first cell loads the runtime, mounts the sonar/ Python package (inlined into this HTML at render time), and primes the chain. It runs once, on first cell evaluation. Every later cell does import sonar like any normal CPython script — the same code that the pytest suite runs.

// The view itself still emits on every drag pixel (so on-screen labels

// only yields on 'change' (mouse release / arrow-key release), so

committed = (view) => {
  const el = (view.querySelector && view.querySelector("input[type=range]")) || view;
  const read = () => (el.valueAsNumber !== undefined ? el.valueAsNumber : el.value);
  return Generators.observe((next) => {
    next(read());
    const h = () => next(read());
    el.addEventListener("change", h);
    return () => el.removeEventListener("change", h);
  });
}

// pyRun: serialised Python execution tied to the calc-status pill.
// Run a block of Python where the last expression must be a JSON
// string; named OJS values are injected into Pyodide globals and the
// parsed JSON is returned. All calls queue onto a single promise so
// globals.set() cannot race between concurrent pyRun invocations.
pyRun = {
  let queue = Promise.resolve();
  const statusEl = document.getElementById('calc-status');
  const textEl = statusEl && statusEl.querySelector('.calc-status__text');
  let pending = 0, total = 0, done = 0;
  const setStatus = (cls, msg) => {
    if (!statusEl || !textEl) return;
    statusEl.classList.remove('calc-status--idle', 'calc-status--running', 'calc-status--done');
    statusEl.classList.add('calc-status--' + cls);
    textEl.textContent = msg;
  };
  const onStart = () => {
    if (pending === 0) { total = 0; done = 0; }
    pending += 1; total += 1;
    setStatus('running', `Running… (${done}/${total} plots)`);
  };
  const onFinish = (ok) => {
    pending = Math.max(0, pending - 1);
    if (ok) done += 1;
    if (pending === 0) setStatus('done', `Complete (${done}/${total} plots)`);
    else setStatus('running', `Running… (${done}/${total} plots)`);
  };
  return (code, vars) => {
    onStart();
    const next = queue.then(async () => {
      const p = await pyodideOjs.pyodidePromise;
      for (const [k, v] of Object.entries(vars || {})) p.globals.set(k, v);
      return p.runPythonAsync(code);
    });
    queue = next.catch(() => {});  // keep queue alive on error
    return next.then(
      (r) => { onFinish(true); return typeof r === 'string' ? JSON.parse(r) : r; },
      (e) => { onFinish(false); throw e; }
    );
  };
}

Step 1 — Transmit waveform

A linear-FM (chirp) pulse at complex baseband:

\[ s(t) = \exp\!\left(j\pi\frac{B}{\tau}\,t^2\right), \qquad t\in[0,\tau). \tag{1}\]

Instantaneous frequency sweeps linearly from 0 to \(B\) over the pulse. The time–bandwidth product \(B\tau\) is the only knob that matters here — it sets every downstream resolution number.

Code 
step1_data = await pyRun(`

Fs = max(4 * B, 2 * fc)

`, {B, tau, fc})

Step 3 — Target echo

A point target at range \(R_0\) moving with radial velocity \(v\) returns a copy of the transmit waveform delayed by \(\tau_d = 2R_0/c\) and shifted in frequency by the (narrow-band) Doppler \(f_d = -2vf_c/c\). Amplitude is set by the radar equation in dB,

\[ \text{level} = \text{SL} - 2\,\mathrm{TL}(R_0, f_c) + \text{TS}. \tag{2}\]

Code 
step3_data = await pyRun(`
import json, numpy as np
from sonar.waveform import lfm_chirp
from sonar.targets import Target, echo
from sonar import C_WATER
Fs = max(4 * B, 2 * fc)
_, s = lfm_chirp(B, tau, Fs)
tgt = Target(R0_m=R0, v_mps=v, TS_dB=-15.0)

})
`, {B, tau, fc, R0, v})

Step 5 — Additive noise + composite received signal

Round it out with white complex-Gaussian noise. The total received signal is \(x(t) = y_\text{echo}(t) + r(t) + n(t)\), with the noise level set so that the input SNR (relative to the echo) matches the slider.

Code 
step5_data = await pyRun(`
import json, numpy as np
from sonar.waveform import lfm_chirp
from sonar.targets import Target, echo
from sonar.reverberation import reverberation
from sonar.noise import white_noise, noise_sigma_for_snr
from sonar import C_WATER
Fs = max(4 * B, 2 * fc)
_, s = lfm_chirp(B, tau, Fs)
n_total = min(int(2.0 * R0 / C_WATER * Fs) + 8192, 16384)
tgt = Target(R0_m=R0, v_mps=v, TS_dB=-15.0)
y_e = echo(s, Fs, fc, tgt, SL_dB=200.0, n_out=n_total)
rng = np.random.default_rng(1)
y_r = reverberation(n_total, Fs, B, decay_tau_s=0.2, rev_level=1e8, rng=rng)
sig_pwr = np.mean(np.abs(y_e[y_e != 0]) ** 2) if np.any(y_e != 0) else 1.0
y_n = white_noise(n_total, noise_sigma_for_snr(sig_pwr, snr_in_dB), rng=rng)
x = y_e + y_r + y_n
w = np.hanning(n_total)
def psd(z):
    Z = np.fft.fft(z * w)
    return np.abs(Z) ** 2 / (Fs * (w * w).sum())
f = np.fft.fftshift(np.fft.fftfreq(n_total, 1.0 / Fs))
Pxx = np.fft.fftshift(psd(x))
Pee = np.fft.fftshift(psd(y_e))
Prr = np.fft.fftshift(psd(y_r))
Pnn = np.fft.fftshift(psd(y_n))
# downsample PSD for plotting
step = max(1, f.size // 2048)
def dB(p): return (10*np.log10(p + 1e-30))[::step].tolist()
json.dumps({
  "f_kHz": (f[::step] / 1e3).tolist(),
  "x": dB(Pxx), "e": dB(Pee), "r": dB(Prr), "n": dB(Pnn),
  "snr_in_dB": float(snr_in_dB),
  "Fs_kHz": float(Fs) / 1e3,
})
`, {B, tau, fc, R0, v, snr_in_dB})

Step 6 — Receive conditioning

In hardware: bandpass to the chirp band, then quadrature-demodulate to complex baseband. Our chain is already at complex baseband, so the “bandpass” here is a baseband lowpass at \(\pm B/2\). The PSD before/after shows the white noise tails getting cut.

Code 
step6_data = await pyRun(`
import json, numpy as np
from sonar.waveform import lfm_chirp
from sonar.targets import Target, echo
from sonar.reverberation import reverberation
from sonar.noise import white_noise, noise_sigma_for_snr
from sonar.receive import baseband_filter
from sonar import C_WATER
Fs = max(4 * B, 2 * fc)
_, s = lfm_chirp(B, tau, Fs)
n_total = min(int(2.0 * R0 / C_WATER * Fs) + 8192, 16384)

y_e = echo(s, Fs, fc, tgt, SL_dB=200.0, n_out=n_total)
rng = np.random.default_rng(2)
y_r = reverberation(n_total, Fs, B, decay_tau_s=0.2, rev_level=1e8, rng=rng)

Step 7 — Matched filter

The centerpiece. Cross-correlate the conditioned signal with the time-reversed conjugate of the transmit pulse. Output peak SNR rises by \(10\log_{10}(B\tau)\) — the only “gain” you get for free in DSP.

Payoff 1. Toggle Apply matched filter in the Controls panel and watch the target peak go from buried-in-noise to unmissable. The plot below uses your current SNR slider; flip the toggle to compare bypass vs. MF on the same axes.

Code 
// Compute BOTH bypass and matched-filter outputs in one Python pass.
// The apply_mf toggle is then a JS-only choice — no pyodide round-trip.
step7_data = await pyRun(`
import json, numpy as np
from sonar.waveform import lfm_chirp
from sonar.targets import Target, echo
from sonar.reverberation import reverberation
from sonar.noise import white_noise, noise_sigma_for_snr
from sonar.receive import baseband_filter
from sonar.matched_filter import matched_filter, mf_gain_dB
from sonar import C_WATER
Fs = max(4 * B, 2 * fc)
_, s = lfm_chirp(B, tau, Fs)
n_total = int(2.5 * 2 * R0 / C_WATER * Fs) + 8192
tgt = Target(R0_m=R0, v_mps=v, TS_dB=-15.0)
y_e = echo(s, Fs, fc, tgt, SL_dB=200.0, n_out=n_total)
rng = np.random.default_rng(3)
y_r = reverberation(n_total, Fs, B, decay_tau_s=0.4, rev_level=1e8, rng=rng)
sig_pwr = np.mean(np.abs(y_e[y_e != 0]) ** 2) if np.any(y_e != 0) else 1.0
y_n = white_noise(n_total, noise_sigma_for_snr(sig_pwr, snr_in_dB), rng=rng)
x_bb = baseband_filter(y_e + y_r + y_n, Fs, B)
mf  = matched_filter(x_bb, s)
range_axis = np.arange(n_total) / Fs * C_WATER / 2.0
def norm_dB(z):
    m = 20 * np.log10(np.abs(z) + 1e-12)
    return (m - m.max()).astype(float)
bp_dB, mf_dB = norm_dB(x_bb), norm_dB(mf)
# Restrict to the display window, then down-sample
r_max = 1.2 * R0 + 200
keep = range_axis <= r_max
range_axis = range_axis[keep]
bp_dB, mf_dB = bp_dB[keep], mf_dB[keep]
step = max(1, range_axis.size // 4000)
json.dumps({
  "range": range_axis[::step].tolist(),
  "bypass": bp_dB[::step].tolist(),
  "mf": mf_dB[::step].tolist(),

How this works

A quick tour of the stack, because the mechanism is the actual point of this document:

  • Quarto authors the page. The .qmd source declares ten sections, eight interactive knobs, and the OJS + pyodide cells that produce each figure.
  • Pyodide is CPython 3.12 compiled to WebAssembly. It runs in a worker thread, ships with NumPy/SciPy as precompiled wheels, and exposes the pyodide.runPythonAsync(...) API that our OJS cells call to do the DSP math. The whole runtime is self-hosted under ./pyodide/ so no CDN request is needed at runtime.
  • OJS (Observable JS) owns the reactive graph. A lightweight pyRun helper wraps each Python call so (a) slider values are piped into Pyodide globals,
    1. calls are serialised onto a single promise queue, and (c) the sticky status pill tracks per-batch progress.
  • Plotting is now client-side. Line, scatter and ROC figures are drawn with Observable Plot; the two range–Doppler heatmaps use Plotly (also self-hosted, no CDN). Python returns bare arrays as JSON, JS renders, and interaction features (hover tooltips, box-zoom, pan, legend toggle) come for free. The Apply matched filter toggle in particular flips in JS without a Python round-trip because both branches (bypass and MF output) are computed once in the data cell.
  • Our DSP code lives in the sonar/ Python package and is imported by both the browser-side Pyodide runtime and the local pytest suite that verifies the analytical invariants (chirp instantaneous frequency, MF gain, MF delay localisation, Doppler-bin localisation, and CA-CFAR’s empirical \(P_\text{fa}\)).

No servers, no notebooks, no kernels, no CDN. The index.html file you are reading is fully self-contained: the Pyodide runtime + scientific wheels + Plotly are all served from the same origin as the page. After the initial ~50 MB warm-up (cached on first visit), the report runs fully offline.

```{ojs}
//| echo: false
// pyRun: serialised Python execution tied to the calc-status pill.
// Run a block of Python where the last expression must be a JSON
// string; named OJS values are injected into Pyodide globals and the
// parsed JSON is returned. All calls queue onto a single promise so
// globals.set() cannot race between concurrent pyRun invocations.
pyRun = {
  let queue = Promise.resolve();
  const statusEl = document.getElementById('calc-status');
  const textEl = statusEl && statusEl.querySelector('.calc-status__text');
  let pending = 0, total = 0, done = 0;
  const setStatus = (cls, msg) => {
    if (!statusEl || !textEl) return;
    statusEl.classList.remove('calc-status--idle', 'calc-status--running', 'calc-status--done');
    statusEl.classList.add('calc-status--' + cls);
    textEl.textContent = msg;
  };
  const onStart = () => {
    if (pending === 0) { total = 0; done = 0; }
    pending += 1; total += 1;
    setStatus('running', `Running… (${done}/${total} plots)`);
  };
  const onFinish = (ok) => {
    pending = Math.max(0, pending - 1);
    if (ok) done += 1;
    if (pending === 0) setStatus('done', `Complete (${done}/${total} plots)`);
    else setStatus('running', `Running… (${done}/${total} plots)`);
  };
  return (code, vars) => {
    onStart();
    const next = queue.then(async () => {
      const p = await pyodideOjs.pyodidePromise;
      for (const [k, v] of Object.entries(vars || {})) p.globals.set(k, v);
      return p.runPythonAsync(code);
    });
    queue = next.catch(() => {});  // keep queue alive on error
    return next.then(
      (r) => { onFinish(true); return typeof r === 'string' ? JSON.parse(r) : r; },
      (e) => { onFinish(false); throw e; }
    );
  };
}
```

## Step 1 — Transmit waveform

A linear-FM (chirp) pulse at complex baseband:

$$
s(t) = \exp\!\left(j\pi\frac{B}{\tau}\,t^2\right), \qquad t\in[0,\tau).
$$ {#eq-chirp}

Instantaneous frequency sweeps linearly from 0 to $B$ over the pulse.
The time–bandwidth product $B\tau$ is the only knob that matters here —
it sets every downstream resolution number.


## Step 3 — Target echo

A point target at range $R_0$ moving with radial velocity $v$ returns a
copy of the transmit waveform delayed by $\tau_d = 2R_0/c$ and shifted in
frequency by the (narrow-band) Doppler $f_d = -2vf_c/c$. Amplitude is set
by the radar equation in dB,

$$
\text{level} = \text{SL} - 2\,\mathrm{TL}(R_0, f_c) + \text{TS}.
$$ {#eq-radareqn}

```{ojs}
//| echo: false
```

## Step 5 — Additive noise + composite received signal

Round it out with white complex-Gaussian noise. The total received signal
is $x(t) = y_\text{echo}(t) + r(t) + n(t)$, with the noise level set so
that the input SNR (relative to the echo) matches the slider.

```{ojs}
//| echo: false
step5_data = await pyRun(`
import json, numpy as np
from sonar.waveform import lfm_chirp
from sonar.targets import Target, echo
from sonar.reverberation import reverberation
from sonar.noise import white_noise, noise_sigma_for_snr
from sonar import C_WATER
Fs = max(4 * B, 2 * fc)
_, s = lfm_chirp(B, tau, Fs)
n_total = min(int(2.0 * R0 / C_WATER * Fs) + 8192, 16384)
tgt = Target(R0_m=R0, v_mps=v, TS_dB=-15.0)
y_e = echo(s, Fs, fc, tgt, SL_dB=200.0, n_out=n_total)
rng = np.random.default_rng(1)
y_r = reverberation(n_total, Fs, B, decay_tau_s=0.2, rev_level=1e8, rng=rng)
sig_pwr = np.mean(np.abs(y_e[y_e != 0]) ** 2) if np.any(y_e != 0) else 1.0

x = y_e + y_r + y_n
w = np.hanning(n_total)

    Z = np.fft.fft(z * w)

f = np.fft.fftshift(np.fft.fftfreq(n_total, 1.0 / Fs))

Pee = np.fft.fftshift(psd(y_e))
Prr = np.fft.fftshift(psd(y_r))
Pnn = np.fft.fftshift(psd(y_n))
# downsample PSD for plotting
```

## Step 6 — Receive conditioning

In hardware: bandpass to the chirp band, then quadrature-demodulate to
complex baseband. Our chain is *already* at complex baseband, so the
"bandpass" here is a baseband lowpass at $\pm B/2$. The PSD before/after
shows the white noise tails getting cut.

```{ojs}

from sonar import C_WATER

_, s = lfm_chirp(B, tau, Fs)
n_total = min(int(2.0 * R0 / C_WATER * Fs) + 8192, 16384)
tgt = Target(R0_m=R0, v_mps=0.0, TS_dB=-15.0)
y_e = echo(s, Fs, fc, tgt, SL_dB=200.0, n_out=n_total)
rng = np.random.default_rng(2)
y_r = reverberation(n_total, Fs, B, decay_tau_s=0.2, rev_level=1e8, rng=rng)
sig_pwr = np.mean(np.abs(y_e[y_e != 0]) ** 2) if np.any(y_e != 0) else 1.0
y_n = white_noise(n_total, noise_sigma_for_snr(sig_pwr, snr_in_dB), rng=rng)
x = y_e + y_r + y_n
x_bb = baseband_filter(x, Fs, B)
w = np.hanning(n_total)
def psd(z):
    Z = np.fft.fft(z * w)
    return np.abs(Z) ** 2 / (Fs * (w * w).sum())
f = np.fft.fftshift(np.fft.fftfreq(n_total, 1.0 / Fs))
step = max(1, f.size // 2048)
def dB(p): return (10*np.log10(np.fft.fftshift(p) + 1e-30))[::step].tolist()
json.dumps({
  "f_kHz": (f[::step] / 1e3).tolist(),
  "before": dB(psd(x)), "after": dB(psd(x_bb)),
  "band_kHz": float(B) / 2e3,
  "Fs_kHz": float(Fs) / 1e3,
})
`, {B, tau, fc, R0, snr_in_dB})
```

## Step 7 — Matched filter {.scrollable}

The centerpiece. Cross-correlate the conditioned signal with the
time-reversed conjugate of the transmit pulse. Output peak SNR rises by
$10\log_{10}(B\tau)$ — the only "gain" you get for free in DSP.

::: {.callout-important appearance="simple"}
**Payoff 1.** Toggle **Apply matched filter** in the Controls panel and
watch the target peak go from buried-in-noise to unmissable. The plot
below uses your current SNR slider; flip the toggle to compare bypass vs.
MF on the same axes.
:::


## Step 9 — Range–Doppler map

Stack $N_\text{pulse}$ consecutive pings along a slow-time axis, FFT
across that axis, and you get a range–Doppler image. Coherent
integration gain is $10\log_{10}N_\text{pulse}$ — a clean 18 dB at $N=64$
versus $N=1$.

::: {.callout-important appearance="simple"}
**Payoff 3.** Set **Pulses integrated** in Controls to 1, then 4, 16, 64.
The target cell rises while its position stays fixed; the noise floor
stays put. That's coherent gain in real time.
:::

```{ojs}
//| echo: false
step9_data = await pyRun(`
import json, numpy as np
from sonar.waveform import lfm_chirp
from sonar.targets import Target, echo
from sonar.noise import white_noise, noise_sigma_for_snr
from sonar.matched_filter import matched_filter
from sonar.range_doppler import range_doppler_map, doppler_axis_velocity
from sonar import C_WATER
Fs = max(4 * B, 2 * fc)
_, s = lfm_chirp(B, tau, Fs)
n_range = int(2.0 * 1.5 * R0 / C_WATER * Fs) + s.size + 64
N_pulse_int = int(N_pulse)
tgt = Target(R0_m=R0, v_mps=v, TS_dB=-15.0)
pulses = np.empty((N_pulse_int, n_range), dtype=np.complex128)
rng = np.random.default_rng(5)
sig_pwr_ref = None
```

## Step 10 — CA-CFAR detection

Cell-averaging CFAR estimates the local noise power from a ring of
training cells (with a small guard band around the cell under test),
multiplies by $\alpha = N\bigl(P_\text{fa}^{-1/N}-1\bigr)$, and thresholds.
We run it across the range axis of the peak Doppler bin and annotate
detections on the RD map.

```{ojs}
//| echo: false

from sonar.range_doppler import range_doppler_map, doppler_axis_velocity

from sonar import C_WATER

N_pulse_int = int(N_pulse)
tgt = Target(R0_m=R0, v_mps=v, TS_dB=-15.0)
pulses = np.empty((N_pulse_int, n_range), dtype=np.complex128)
rng = np.random.default_rng(6)
sig_pwr_ref = None
```

## How this works {.unnumbered}

A quick tour of the stack, because the *mechanism* is the actual point of
this document:

- **[Quarto](https://quarto.org)** authors the page. The `.qmd` source
  declares ten sections, eight interactive knobs, and the OJS + pyodide
  cells that produce each figure.
- **[Pyodide](https://pyodide.org)** is CPython 3.12 compiled to
  WebAssembly. It runs in a worker thread, ships with NumPy/SciPy as
  precompiled wheels, and exposes the `pyodide.runPythonAsync(...)`
  API that our OJS cells call to do the DSP math. The whole runtime is
  self-hosted under `./pyodide/` so no CDN request is needed at runtime.
- **[OJS](https://quarto.org/docs/interactive/ojs/)** (Observable JS)
  owns the reactive graph. A lightweight `pyRun` helper wraps each
  Python call so (a) slider values are piped into Pyodide globals,
  (b) calls are serialised onto a single promise queue, and (c) the
  sticky status pill tracks per-batch progress.
- **Plotting is now client-side.** Line, scatter and ROC figures are
  drawn with [Observable Plot](https://observablehq.com/plot); the two
  range–Doppler heatmaps use [Plotly](https://plotly.com/javascript/)
  (also self-hosted, no CDN). Python returns bare arrays as JSON, JS
  renders, and interaction features (hover tooltips, box-zoom, pan,
  legend toggle) come for free. The **Apply matched filter** toggle in
  particular flips in JS without a Python round-trip because both
  branches (bypass and MF output) are computed once in the data cell.
- **Our DSP code** lives in the `sonar/` Python package and is imported
  by both the browser-side Pyodide runtime and the local `pytest` suite
  that verifies the analytical invariants (chirp instantaneous frequency,
  MF gain, MF delay localisation, Doppler-bin localisation, and
  CA-CFAR's empirical $P_\text{fa}$).

No servers, no notebooks, no kernels, no CDN. The `index.html` file you
are reading is fully self-contained: the Pyodide runtime + scientific
wheels + Plotly are all served from the same origin as the page. After
the initial ~50 MB warm-up (cached on first visit), the report runs
fully offline.
